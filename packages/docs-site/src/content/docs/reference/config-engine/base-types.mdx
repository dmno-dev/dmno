# Base Types

DMNO comes with a comprehensive set of base types to cover the majority of use cases. They can also be extended to create more application specific types as needed. See [_creating your own types_](/reference/config-engine/base-types/#creating-your-own-types) for more on this. 

These types are used when defining

```javascript

// example use for creating a type

const myType = DmnoBaseTypes.String({ settingsObject });
```


## `String`

DMNO Built-in data type for Strings. Includes the following (optional) settings: 

```
minLength?: number;
maxLength?: number;
isLength?: number;
startsWith?: string;
endsWith?: string;
matches?: RegExp | string;
```

Examples: 
```javascript
const myMinMaxType = DmnoBaseTypes.String({ minLength: 2, maxLength: 5 });
const myFixedLengthType = DmnoBaseTypes.String({ isLength: 5 });
const myStartsWithType = DmnoBaseTypes.String({ startsWith: 'pk_' });

```

Example Type Errors:
```typescript
const myStartsWithType = DmnoBaseTypes.String({ startsWith: 'pk_' });

```



## `Number`

DMNO Built-in data type for Numbers. Includes the following (optional) settings: 

```
min?: number;
max?: number;
```

Examples: 
```javascript
const myMinMaxType = DmnoBaseTypes.String({ min: 2, max: 5 });

```

## Creating your own types

:::tip[Advanced]
  In the event that our base types aren't sufficient, you can extend them to create your own, more specific types using `createDmnoDataType`. 
:::


### `createDmnoDataType(opts)` 

Creates a new DMNO data type. 

```javascript
const myType = createDmnoDataType({
  name: 'MyType',
  extends: DmnoBaseTypes.String({
    // settings object
  }),
  settingsSchema: {
    // user specific settings object
  },
  validate: (value) => {
    // return true if value is valid
  },
  coerce: (value) => {
    // return coerced value
  },
});
```

You can then use it in your config schema like so: 

```javascript
const myType = DmnoBaseTypes.MyType({ settingsObject });

export default defineConfigSchema({
  name: 'MyConfig',
  parent: 'root',
  schema: {
    MYFIELD: {
      extends: myType,
      required: true,
    },
    MYFIELD2,
    MYFIELD3,
  },
});
```
